# v25.8 to v25.9
## Short Migration Note

## What changed
v25.9 added an observer-only governance diagnostics layer and a bounded same-shock governance comparison surface.

## What stayed the same
The public claim is that the governance layer is diagnostic. It does not rewrite the core visible-stress engine into a full new causal governance model.

## Public outputs added or stabilized
- `epistemic_integrity_proxy`
- `decision_friction_proxy`
- `platform_dependency_proxy`
- `mosaic_response_proxy`
- `governance_erosion_score`
- bounded comparison artifacts for higher-friction vs reduced-friction conditions

## What this migration does not justify
- forecasting claims,
- fully calibrated external probability claims,
- claims that v25.9 completed governance integration,
- claims that the public build already contains first-class fractal diagnostics.

## Why it matters
This migration makes it possible to compare visible stress and hidden deterioration within the same documented public release.
