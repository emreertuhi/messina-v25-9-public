# Relation to the Article
## How This Repository Supports "The Strait Is Not the Crisis"

## Purpose

This file explains how the public repository relates to the policy article and where the strongest public support for the article's claims is located.

## Core relation

The article is broader in prose, chronology, and policy interpretation than the executable public package. The repository should not be read as proving every sentence in the article. It should be read as exposing the bounded public comparison surface behind the article's core governance argument.

The article-facing claim supported by this repository is narrow:

> Under documented governance overlays, visible stress can remain broadly similar while governance diagnostics deteriorate materially under reduced-friction conditions.

## What the repository directly supports

The repository directly supports the claims that:

- similar opening shocks can generate materially different governability outcomes,
- visible stress and control quality can diverge,
- reduced-friction conditions can worsen governance diagnostics even when peak stress moves only slightly,
- platform dependency can rise as internal braking architecture thins,
- policy-facing concern should focus on second-order governability rather than only first-contact intensity.

## What the repository does not directly support

The repository does not claim to provide:

- the full historical internal engine,
- an executable proof for every policy proposal in the article,
- a complete legal simulation of War Powers or congressional procedure,
- a causal Decision Sovereignty layer,
- a public fractal-diagnostics stack,
- public execution of all 24 internal module families.

## Article wording guardrail

When linking the repository from the article, use this safer formulation:

> The public repository exposes the governance-observer formulas, configs, comparison artifacts, and methodology notes behind this claim.

Avoid saying simply:

> The working code is public.

That phrasing is too broad because the public repo does not expose the full internal v25.x engine.
