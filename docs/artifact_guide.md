# Artifact Guide
## Public Evidence Surface for MESSINA v25.9

This guide explains which public artifacts are primary evidence for the current release and how they should be read.

## Primary evidence files

### `artifacts/v25.9/full_run_report.md`
Use this as the release-level summary of the current public build.

### `artifacts/v25.9/governance_signals.json`
Use this as the structured baseline governance-diagnostic output bundle.

### `artifacts/v25.9/single_run.json`
Use this as the structured baseline public run output.

### `artifacts/v25.9/governance_comparison.json`
Use this as the structured same-shock / different-governance comparison output.

### `artifacts/v25.9/governance_summary.md`
Use this as the human-readable comparison summary.

### `artifacts/v25.9/test_results.json`
Use this to verify that the public-safe runner and its bounded outputs passed the included validation tests in this release package.

### `version_notes/v25_8_to_v25_9_short.md`
Use this to understand what v25.9 added and what it did not add.

## Reading rule

These artifacts support narrow build claims about the documented public release. They do not, by themselves, justify forecasting claims, universal geopolitical claims, or claims about the full historical internal engine.


## Scope note
For the formal release boundary and article-repo relationship, see `docs/release_scope.md` and `docs/article_relation.md`.


## Overlay-suite evidence
The following files are supporting public evidence for no-code comparison expansion:
- `artifacts/overlay_suite/friction_spectrum_summary.md`
- `artifacts/overlay_suite/policy_overlay_summary.md`
- `artifacts/overlay_suite/theme_overlay_summary.md`

These do not replace the core v25.9 files. They extend the comparison surface in a bounded way.
