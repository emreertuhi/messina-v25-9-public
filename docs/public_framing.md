# Public Framing Guide

## Single most important rule

Do not let the public-facing description outrun the current build.

## Safest one-sentence description

**MESSINA is a mechanism-encoding, research-grade scenario-exploration framework whose outputs should be read as comparative stress diagnostics under encoded assumptions.**

## Safest short paragraph

**MESSINA is a non-predictive framework for exploring how chokepoint crises propagate through tightly coupled systems under explicit assumptions. It does not forecast real-world events or assign reliable real-world probabilities. Instead, it compares how crisis pathways, control quality, and hidden institutional deterioration behave across structured conditions.**

## Preferred labels
- comparative stress-testing framework
- mechanism-encoding framework
- research-grade scenario-exploration framework
- non-predictive scenario model
- circulation-crisis stress-test model

## Use with care
- simulation framework
- crisis-modeling framework
- structured scenario engine

## Avoid
- forecasting engine
- predictive model
- geopolitical AI
- decision engine
- real-world probability machine
- automated policy system

## How to talk about v25.9

Preferred phrasing:
- current public-facing baseline
- bounded public release
- observer-only governance diagnostic stage
- current public-repo center of gravity

Do not use:
- full governance engine
- completed sovereignty layer
- final architecture
- predictive release

## Safe sentence for the repository

> This public repository is centered on **v25.9**, the current public-facing baseline of the framework. It provides a bounded, publication-safe comparison surface rather than a full release of the historical internal engine.


## Scope reminder
When speaking publicly about the repository, use the release boundary stated in `docs/release_scope.md` and avoid implying that the public package contains the project's full historical internal engine.

- complete scenario family release
