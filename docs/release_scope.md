# Release Scope
## What This Public Repository Does and Does Not Release

## Purpose

This file defines the public release boundary for MESSINA v25.9 public-safe v8.

The associated working paper describes a broader internal v25.x design lineage. This repository does not expose that entire lineage as executable code. It releases the bounded governance-observer comparison surface needed to inspect the article-facing claim that visible stress and governability can diverge under different institutional-friction conditions.

## Current release boundary

This repository releases:

- public theory and methodology notes,
- public limits and interpretation rules,
- a bounded public-safe runner,
- public scenario configs,
- public governance-observer formulas,
- public formula and parameter registries,
- public artifacts for the v25.9 governance-observer surface,
- validation tests for the bounded public runner,
- friction-spectrum, policy-overlay, and theme-overlay summaries,
- non-executable scenario briefs for future cross-theater comparison.

## Broader architecture versus public release

The broader internal v25.x architecture is documented as a 72-variable, six-layer, two-dozen-module framework. That description is an architectural documentation claim.

The public repository claim is narrower. The executable public claim is that the included configs, formulas, runner, and artifacts reproduce the documented governance-observer comparison surface.

The public release should therefore be read as:

> an inspectable public governance-observer subset of MESSINA v25.9, not a complete release of the historical internal engine.

## What the public release supports

The public release supports these narrow claims:

- MESSINA is non-predictive and mechanism-encoding in its research design.
- The public v25.9 release exposes observer-only governance diagnostics.
- Under documented comparison configs, visible peak stress can remain broadly similar while governance diagnostics worsen under reduced-friction conditions.
- Public formulas, configs, artifacts, and tests are inspectable and reproducible in bounded form.
- Policy interpretations should be read as protocol-level implications, not direct outputs of a legal or command model.

## What the public release does not support

The public release does **not** claim:

- full external empirical calibration,
- rare-event probability estimation,
- full historical-engine replay,
- executable reproduction of all 24 internal module families,
- a complete causal governance layer,
- a public Decision Sovereignty dynamics layer,
- a public fractal-diagnostics stack,
- live-data early warning.

## Why the release is bounded

The release is bounded to keep public claims aligned with public evidence. This prevents the repository from asking readers to trust non-public module behavior or internal design memory. It also keeps the policy article, working paper, and public code at the same claim level.

## Safe reading rule

Use this repository to inspect the v25.9 public governance-observer surface. Do not treat it as a full disclosure of every internal architecture branch or as a forecasting model.
